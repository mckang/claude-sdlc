# PRD: 로컬 저장 Todo 관리 웹앱

- **식별자**: todo-app
- **작성일**: 2026-04-23
- **작성자**: PM (T'Challa)
- **버전**: 1.0
- **상태**: approved

## 1. 배경·문제 정의

개인 생산성 도구가 넘쳐나지만, 대부분은 **회원가입·동기화 설정**을 요구한다. "그냥 브라우저 탭 하나 열어 잠깐 적어두고, 새로고침해도 남아 있으면 되는" 경량 Todo가 의외로 드물다.

이 앱은 **2차 목적이 더 크다**: Claude Code SDLC 플러그인의 전체 워크플로우를 따라가 완성한 **학습용 레퍼런스**. 초심자가 이 예제의 `docs/` 트리만 읽어도 "SDLC 커맨드들이 어떤 산출물을 남기는지" 감을 잡는다.

## 2. 목표 · 성공 지표

| 지표 | 기준 |
|---|---|
| 1차 사용자가 첫 Todo 를 적기까지 | ≤ 5초 (랜딩 → 입력 포커스 → 저장) |
| localStorage 영속성 | 새로고침·탭 닫고 재오픈 후 0 건 손실 |
| 라이트하우스 Performance | ≥ 90 (모바일) |
| 코드 라인 수 | ≤ 500 LOC (테스트 제외) |
| 학습용 가치 | 예제만 보고 다른 feature 의 SDLC 산출물을 작성할 수 있다 |

## 3. 타겟 사용자

- **P1 페르소나**: 브라우저 북마크 수준의 메모 도구를 원하는 개인
- **P2 페르소나** (2차 목적): Claude Code SDLC 플러그인을 처음 배우는 개발자

## 4. 사용자 스토리 / 유스케이스

- US-1: 사용자로서, 랜딩하자마자 입력창에 포커스된 상태로 바로 입력하고 싶다.
- US-2: 사용자로서, Enter 키로 추가하고 입력창이 즉시 비워지길 원한다.
- US-3: 사용자로서, 체크박스로 완료 표시를 하면 시각적으로 구분되길 원한다.
- US-4: 사용자로서, 새로고침 해도 내가 적은 Todo 가 그대로 있길 원한다.
- US-5: 사용자로서, 완료한 것을 숨기거나 미완료만 보고 싶다.
- US-6: 사용자로서, 내가 해야 할 일이 몇 건 남았는지 한눈에 알고 싶다.

## 5. 기능 요구사항 (FR)

| ID | 요구 |
|---|---|
| FR-1 | 입력 폼: 빈 문자열·공백만 입력 방지, 최대 200자 제한 |
| FR-2 | 추가 즉시 리스트 상단에 반영 (최신이 위) |
| FR-3 | 체크박스로 완료 토글 (체크 시 취소선 + 회색) |
| FR-4 | 항목당 삭제 버튼 |
| FR-5 | localStorage 키 `todo-app:v1:items` 에 JSON 배열 저장 |
| FR-6 | 필터 3종: all · active · completed (라디오 스타일 탭) |
| FR-7 | 하단 status bar: "<N>개 남음" · 선택된 필터 · (선택) "완료 <M>개 지우기" 버튼 |
| FR-8 | Todo 0건일 때 안내 문구 ("아직 할 일이 없습니다. 위에서 추가하세요.") |

## 6. 비기능 요구사항 (NFR)

| 축 | 기준 |
|---|---|
| 성능 | 초기 JS 번들 ≤ 100 KB gzipped, First Contentful Paint ≤ 1.5s (4G) |
| 접근성 | 모든 interactive 요소에 label · role · focus ring. 키보드 전용 조작 가능 |
| 호환성 | 최신 Chrome · Firefox · Safari · Edge. IE 미지원 |
| 보안 | localStorage 외 저장소 사용 금지. XSS 방어 (React 기본 escape 에 의존) |
| 관찰성 | `console.error` 레벨 이상만 허용. 사용자 행동 추적 없음 |

## 7. 수용 기준 (AC)

PRD 수준의 대표 AC (Story 단위 AC 는 Plan 에서 세분화):

- **AC-A**: 랜딩 → 입력 → Enter → 새로고침 → 항목이 그대로 남아 있다
- **AC-B**: 체크박스 클릭 → 즉시 시각적 변화 → localStorage 에 반영 → 새로고침 후 유지
- **AC-C**: 필터 전환 시 URL 은 바뀌지 않고 리스트만 필터링된다 (client-only)
- **AC-D**: 빈 상태에서 안내 문구가 보이고, 1건 추가 즉시 사라진다

## 8. 범위 외 (Out of Scope)

- 서버·DB·API
- 사용자 계정·인증·동기화
- 기기 간 동기
- 태그·카테고리·우선순위·마감일
- 드래그 정렬
- 오프라인 PWA (localStorage 만으로도 오프라인 동작하지만 Service Worker 설치는 안 함)

## 9. 가정 · 미결 사항

- 가정: 사용자는 모던 브라우저(2022 이후)를 사용한다
- 가정: localStorage 용량 제한 (5-10MB) 에 도달하지 않는다 — Todo 수가 수만 건을 넘지 않는다
- 가정: 단일 탭·단일 사용자. 동시 오픈 탭 간 동기화는 고려하지 않음 (동작하긴 함 — React state 와 localStorage 가 desync 되면 새로고침 시 localStorage 쪽이 우선)
