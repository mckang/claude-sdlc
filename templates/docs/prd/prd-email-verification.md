# PRD: 사용자 이메일 인증 (v1)

- **작성일**: 2026-04-20
- **상태**: approved
- **우선순위**: P1

## 1. 배경 및 문제

현재 가입 시 이메일 유효성을 형식만 검증하고 실제 소유 여부는 확인하지 않는다. 이 때문에:
- 가짜 이메일로 가입된 계정이 전체의 8% 수준
- 비밀번호 재설정 등 이메일 기반 기능이 실패
- 영업팀이 리드 이메일에 접근 불가 (사용자 신뢰도 문제)

## 2. 목표 및 성공 지표

- **주 지표**: 인증 완료율 70% 이상 (가입 후 24시간 내)
- **부 지표**:
  - 가짜 계정 비율을 1% 이하로 감소
  - 비밀번호 재설정 성공률 95% → 99%
- **출시 목표**: 6주 이내

## 3. 유저 스토리

- **US-1**: 가입자로서 이메일로 받은 인증 링크를 클릭해 계정을 활성화하고 싶다
- **US-2**: 가입자로서 인증 메일을 받지 못한 경우 재전송을 요청하고 싶다
- **US-3**: 가입자로서 인증 링크가 만료되어도 새로 받을 수 있어야 한다
- **US-4**: 운영자로서 장기간 미인증 계정을 조회하고 정리할 수 있어야 한다

## 4. 기능 요구사항 (FR)

- **FR-1**: 가입 완료 시 자동으로 인증 메일 발송
- **FR-2**: 메일에 포함된 링크 클릭 시 계정 상태 `unverified` → `verified` 변경
- **FR-3**: 인증 링크는 단회용, 24시간 유효
- **FR-4**: 사용자가 "인증 메일 재전송" 요청 가능 (분당 1회, 시간당 5회 제한)
- **FR-5**: 미인증 계정은 로그인 허용하되 주요 기능 제한 (결제·게시물 작성 차단)
- **FR-6**: 관리자 대시보드에 "7일 이상 미인증 계정 목록" 페이지 추가

## 5. 비기능 요구사항 (NFR)

- **성능**: 인증 메일 발송 지연 30초 이내, 링크 클릭 시 반영 p95 500ms
- **보안**:
  - 토큰은 URL에 노출되지만 단회용이며 24시간 후 자동 무효
  - 토큰은 cryptographically secure random (최소 128비트 엔트로피)
  - 인증 재전송은 IP·이메일 복합 Rate limit
- **관찰성**: 인증 완료율을 일 단위로 대시보드에 집계
- **호환성**: 주요 이메일 클라이언트 (Gmail·Outlook·Apple Mail·Naver) 링크 동작 확인
- **국제화**: 한국어·영어 템플릿 제공 (사용자 언어 설정 따름)

## 6. 수용 기준 (Acceptance Criteria)

- **AC-1**: 가입 후 30초 이내 인증 메일 수신 (정상 경로)
- **AC-2**: 링크 클릭 후 "인증 완료" 페이지 표시, 자동 로그인 + 대시보드 이동
- **AC-3**: 만료된 링크 클릭 시 "링크가 만료되었습니다" 페이지 + "재전송" 버튼 제공
- **AC-4**: 재전송 요청을 분당 1회 초과 시 429 에러
- **AC-5**: 미인증 계정이 결제 시도 시 인증 유도 모달 표시
- **AC-6**: 관리자가 미인증 계정 목록을 페이지당 50개로 조회 가능

## 7. 엣지 케이스

- 이메일 발송 실패 (SMTP 장애) — 재시도 3회 + 관리자 알림
- 토큰 충돌 (매우 낮은 확률) — 재생성
- 사용자가 이메일 주소 변경 — 기존 토큰 무효화 + 신규 발송
- 메일 서비스의 스팸 필터링 — SPF/DKIM/DMARC 설정 필수
- 사용자가 여러 기기에서 동시 인증 시도 — 첫 번째만 성공, 이후는 "이미 인증됨"

## 8. 가정 및 미결 사항

- 이메일 발송 서비스는 AWS SES 사용 가정 (인프라팀 확인 필요)
- 관리자 대시보드는 기존 어드민 UI 프레임워크 재사용
- **미결**: 인증 전 로그인 완전 차단 vs 부분 허용 — 현재는 부분 허용으로 결정됐으나 UX팀 재검토 필요

## 9. 범위 외 (Out of Scope)

- SMS 인증
- 소셜 로그인 (OAuth) 계정의 이메일 인증 (해당 제공자 인증 신뢰)
- 이메일 주소 변경 기능 자체 (별도 기능)
- 미인증 계정 자동 삭제 (운영자 수동 관리)
