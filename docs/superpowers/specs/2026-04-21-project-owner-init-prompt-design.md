# 프로젝트 오너 이름을 `/sdlc:init` 에서 대화식으로 수집

- **작성일**: 2026-04-21
- **상태**: Draft (사용자 리뷰 대기)

## 배경 / 동기

`sdlc` 플러그인의 `commands/meeting.md` 두 곳(31행, 190행)에 플러그인 작성자의 이름(`Thomas`)이 하드코딩되어 있다. 플러그인을 배포하면 다른 사용자 환경에서도 "내(Thomas)에게 물어봐"라는 지시가 그대로 실행되어 모든 프로젝트에서 `Thomas`가 의사결정자로 간주된다.

프로젝트 오너의 이름을 **각 프로젝트의 `CLAUDE.md`** 에서 읽어오도록 변경하고, `/sdlc:init` 실행 시 대화식으로 입력받아 자동 기입한다. `CLAUDE.md`는 세션 시작 시 자동 로드되므로 `meeting` 커맨드에서 별도 Read 없이 오너 이름을 알 수 있다.

## 스코프

- **In scope**
  - `templates/CLAUDE.md` 에 "프로젝트 오너" 섹션 추가 (이름 플레이스홀더)
  - `commands/init.md` 5단계 재설계 (대화식 프롬프트 + 조건부 CLAUDE.md 생성/append)
  - `commands/meeting.md` 하드코딩된 `Thomas` 제거 → "프로젝트 오너" 참조로 교체
  - `plugin.json`, `marketplace.json` 패치 버전 동시 bump
- **Out of scope**
  - 기존 프로젝트의 이미 설치된 CLAUDE.md 자동 마이그레이션 도구
  - 오너 역할·권한 확장 (지금은 "최종 의사결정자" 한 가지 용도)
  - 다국어 프롬프트 (한국어만)

## 설계

### 1. `templates/CLAUDE.md` 변경

파일 **맨 끝**에 다음 섹션 추가. 이름은 `{NAME}` 플레이스홀더로 두되, `/sdlc:init` 이 사용자 입력으로 치환.

```markdown
---

## 프로젝트 오너
- **이름**: {NAME}
- **역할**: 최종 의사결정자. 미팅·토론 중 결정이 필요하면 반드시 이 사용자에게 질의.
```

오너 섹션을 **하단**에 배치하는 이유: 기존 CLAUDE.md에 append 하는 경로와 위치가 동일해 구현이 단순해지고, 사용자가 수동으로 수정할 때도 찾기 쉬움.

### 2. `commands/init.md` 5단계 재설계

현재 단순 `cp` 로직을 다음 흐름으로 교체한다.

**2-a. 항상 이름을 먼저 묻는다**

init 실행 초반(CLAUDE.md 처리 전)에 대화식 프롬프트 출력:

> 이 프로젝트의 오너(최종 의사결정자) 이름을 알려주세요. 미팅·토론에서 결정이 필요할 때 Claude가 이 사람에게 질의합니다. (엔터만 치면 스킵)

사용자 응답을 쉘 변수(`OWNER_NAME`)에 저장. 빈 응답이면 이후 모든 오너 섹션 처리 스킵.

**2-b. CLAUDE.md 없을 때**

기존 로직대로 `cp` 로 템플릿 복사 후, 이름이 있으면 템플릿의 `{NAME}` 을 `sed` 로 치환. 이름이 없으면 오너 섹션 자체를 삭제 (헤더의 `---` 구분선 포함).

**2-c. CLAUDE.md 있을 때**

기존 파일에 `## 프로젝트 오너` 헤더가 이미 있는지 `grep -q` 로 확인.
- **없으면**: 이름이 주어진 경우 파일 끝에 구분선과 섹션을 append.
- **있으면**: "이미 오너 섹션이 있습니다. 수동 확인 권장." 안내 출력, append 생략.

### 3. `commands/meeting.md` 하드코딩 제거

- **31행**
  - Before: `참석자에는 적극적으로 대화에 참여하지는 않지만 필요한 의사결정을 내릴수 있는 내(Thomas)가 들어가 있어.`
  - After: `참석자에는 적극적으로 대화에 참여하지는 않지만 필요한 의사결정을 내릴 수 있는 프로젝트 오너(CLAUDE.md 의 "프로젝트 오너" 섹션 참조)가 포함된다. 오너 섹션이 없으면 현재 사용자를 오너로 간주한다.`
- **190행**
  - Before: `의사 결정이 필요한 부분이 있으면 가정하지 말고 대화도중에라도 반드시 나(Thomas)에게 물어보고 진행해줘`
  - After: `의사 결정이 필요한 부분이 있으면 가정하지 말고 대화 도중에라도 반드시 프로젝트 오너에게 물어보고 진행해줘.`

`CLAUDE.md` 는 세션 시작 시 자동 로드되므로 오너 이름은 이미 컨텍스트에 있음 → 추가 Read 불필요.

### 4. 버전 bump

- `plugin.json`: `1.0.1` → `1.0.2`
- `marketplace.json`: 동일 버전 동기화

```
이 프로젝트의 오너(최종 의사결정자) 이름을 알려주세요. 미팅·토론에서
결정이 필요할 때 Claude가 이 사람에게 질의합니다. (엔터만 치면 스킵)
> Thomas Kang

✓ docs/ 트리 생성
✓ 표준 문서 25개 설치
✓ 가이드 설치
✓ 샘플 PRD/아키텍처 설치
✓ CLAUDE.md 생성 완료 (오너: Thomas Kang)
✓ .gitignore 업데이트
```

기존 CLAUDE.md가 있고 오너 섹션이 없는 경우:

```
✓ 기존 CLAUDE.md 유지 — 파일 끝에 "## 프로젝트 오너" 섹션 append 완료 (오너: Thomas Kang)
```

기존 CLAUDE.md가 있고 오너 섹션이 이미 있는 경우:

```
⚠️  기존 CLAUDE.md에 이미 "## 프로젝트 오너" 섹션이 있습니다. 수동 확인 권장.
```

## 엣지 케이스 / 에러 처리

| 상황 | 처리 |
|------|------|
| 사용자가 엔터만 치고 스킵 | 오너 섹션 추가 안 함. CLAUDE.md는 오너 섹션이 없는 상태로 생성 (템플릿의 `---` 구분선과 섹션 블록 제거). meeting.md는 "섹션 없으면 현재 사용자를 오너로 간주"로 동작. |
| 이름에 특수문자(`/`, `&`, `"` 등) 포함 | `sed` 치환 시 구분자를 `\|` 로 쓰고 입력값 이스케이프. |
| 기존 CLAUDE.md에 오너 섹션이 이미 있음 | append 하지 않음. 경고만 출력. |
| `/sdlc:init` 을 재실행 | CLAUDE.md가 이미 있으므로 기존 정책(덮어쓰지 않음)대로 동작. 오너 섹션은 위 규칙 따름. |

## 테스트 시나리오 (수동)

1. **신규 프로젝트 + 이름 입력** → CLAUDE.md 생성, 하단에 오너 섹션, 이름 올바름.
2. **신규 프로젝트 + 이름 스킵** → CLAUDE.md 생성, 오너 섹션 없음, 끝에 불필요한 `---` 없음.
3. **기존 CLAUDE.md (오너 섹션 없음) + 이름 입력** → 파일 끝에 섹션 append, 기존 내용 보존.
4. **기존 CLAUDE.md (오너 섹션 있음) + 이름 입력** → append 안 함, 경고 출력.
5. **meeting 커맨드 실행** → Claude 가 CLAUDE.md 의 오너 이름을 인용해서 질의.
